# Состояние проекта и применённые решения

> Архивный файл. Исторические решения ниже больше не считаются каноническим источником истины и могут расходиться с текущим кодом.

## Где теперь канон

- запуск и локальная работа: `README.md` и `RUN.md`
- ручная проверка после правок: `QA_CHECKLIST.md`
- турнирные инварианты и логика: `TOURNAMENT_LOGIC.md`
- стоимость и история шагов: `.cursor/project-cost-tracking.md`

## Текущее краткое состояние

- основной dev-режим: `npm run dev:live`
- единый локальный CI-equivalent прогон: `npm run verify:ci`
- backend e2e больше не держится на старом `jest-e2e` каркасе; вместо этого используется runtime-check через `backend/src/scripts/e2e-health-check.ts`
- production deploy канонически идёт через `scripts/deploy-prod.sh` и root scripts `deploy:prod` / `deploy:prod:frontend`

## Зачем файл оставлен

- чтобы не терять исторический контекст по старым решениям
- чтобы было понятно, что прошлые заметки нужно сверять с текущим runtime

Если нужен актуальный статус проекта, ориентироваться нужно не на этот архив, а на текущий код, README/RUN и verification scripts.
