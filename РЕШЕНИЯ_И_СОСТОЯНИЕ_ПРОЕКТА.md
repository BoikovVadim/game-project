# Состояние проекта и применённые решения

Документ фиксирует все изменения по запросам, чтобы не начинать заново.

---

## Как запустить

Из **корня проекта** (папка «Игра»):

```bash
npm run dev:live
```

- Фронт: **http://localhost:3000**
- Бэкенд: **http://localhost:3001** (прокси с фронта)

Если нет Node в PATH, сначала: `nvm use 20` (или ваша версия).

---

## Главная страница

- **URL:** http://localhost:3000 или http://localhost:3000/#/
- **Поведение:**
  - Вверху справа — тёмная шапка с кнопками **«Регистрация»** и **«Вход»** (если не залогинен) или **«Личный кабинет»** (если залогинен).
  - В центре — блок «Главная» с текстом и кнопками «Вход» и «Регистрация» (компонент `Home`).
- **Файлы:** `Frontend/src/App.tsx` (ветка `isRoot`), `Frontend/src/components/Home.tsx`, `Frontend/public/index.html` (hash `#/`, заглушка в `#root`).

---

## Вход и регистрация

- **Вход:** http://localhost:3000/#/login — компонент `Login.tsx`.
- **Регистрация:** http://localhost:3000/#/register — компонент `Register.tsx`.
- После успешного входа редирект идёт на **главную** (`/`), а не в профиль (чтобы не показывать белый экран).

---

## Личный кабинет (профиль)

- **URL:** http://localhost:3000/#/profile
- Чтобы не было белого экрана, по маршруту `/profile` показывается **простая** страница кабинета (`ProfilePlaceholder`), а не тяжёлый `Profile`.
- На ней: заголовок «Личный кабинет», загрузка email с API `/users/profile`, кнопки «На главную» и «Выйти».
- **Файл:** `Frontend/src/components/ProfilePlaceholder.tsx`, подключение в `App.tsx` (Route `/profile`).

---

## Игры по прямой ссылке

- **URL:** http://localhost:3000/#/games
- Отдельный маршрут `/games` с `forceSection="games"`; при открытии показывается кабинет с разделом «Игры».
- В `Profile.tsx` есть поддержка пропа `forceSection`.

---

## Сборка и скрипты

- **Backend:** в `backend/package.json` скрипт `build`: `node node_modules/typescript/bin/tsc` (чтобы сборка работала при локальном TypeScript).
- **Root:** в `package.json` скрипт `dev:live`: после сборки backend выполняется `cd ..`, затем из корня запускается `npx concurrently` с двумя процессами (`sh -c 'cd backend && ...'` и `sh -c 'cd Frontend && npm run start'`), чтобы и бэкенд, и фронт работали из своих папок.

---

## Краткая схема маршрутов

| Путь        | Не залогинен     | Залогинен              |
|------------|------------------|------------------------|
| `/`        | Главная (Home)   | Главная «Вы вошли» + ссылка в кабинет |
| `/login`   | Страница входа   | —                      |
| `/register`| Регистрация      | —                      |
| `/profile` | Редирект на login| ProfilePlaceholder     |
| `/games`   | Редирект на login| Profile с forceSection games |

---

## Важные файлы

- `Frontend/src/App.tsx` — роутинг, главная (isRoot), шапка, ProfilePlaceholder.
- `Frontend/src/components/Home.tsx` — главная с кнопками Вход/Регистрация.
- `Frontend/src/components/ProfilePlaceholder.tsx` — простая страница кабинета.
- `Frontend/src/components/Login.tsx` — после входа `navigate('/')`.
- `Frontend/public/index.html` — установка `#/` при пустом hash, заглушка главной в `#root`.
- `package.json` (корень) — `dev:live`.
- `backend/package.json` — `build` через `node node_modules/typescript/bin/tsc`.

Проект восстановлен по всем описанным решениям; при необходимости можно опираться на этот файл.
