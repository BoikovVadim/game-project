# Перенос проекта на другой компьютер

> Legacy-заметка: документ описывает старый сценарий эпохи `backend/db.sqlite` и запуска через `npm run dev`. Для текущего проекта он не считается каноническим guide без дополнительной актуализации под действующий backend/runtime.

## Что нужно перенести

1. **Исходный код и база данных** — всё в Git (включая `backend/db.sqlite` с пользователями и прогрессом)

---

## Шаг 1: На текущем компьютере

### Код — отправить в Git

```bash
git add .
git commit -m "Сохраняю изменения"
git push origin main
```

(Если ветка называется `master`, замените `main` на `master`.)

### База данных

База `backend/db.sqlite` хранится в репозитории и уйдёт в GitHub вместе с `git push`.

---

## Шаг 2: На новом компьютере

### 1. Клонировать или скопировать проект

```bash
git clone <URL_вашего_репозитория> game-project-main
cd game-project-main
```

Если репозиторий ещё не создан — скопируйте папку проекта с текущего компьютера (без `node_modules`).

### 2. Установить зависимости

```bash
npm install
```

### 4. Запустить проект

```bash
npm run dev
```

Откройте http://localhost:3000

---

## Краткий чеклист

- [ ] `git push` с текущего компьютера
- [ ] `git pull` на новом компьютере (или `git clone`)
- [ ] `npm install`
- [ ] `npm run dev`
